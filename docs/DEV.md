# 前端页面设计文档 V2 — 悬浮管家与日历工作台

## 一、 整体页面布局 (The Planner Layout)

全面对标 reclaim.ai 的空间划分，采用经典且高效的三栏/两栏自适应布局，确保日历占据视觉核心。

- **左侧边栏 (Left Sidebar - 导航与过滤)**
  - 包含全局导航（日历、任务、偏好设置）。
  - 日历视图切换（日、周、月）。
  - **视觉约束**：固定宽度（如 240px），可折叠，背景色略深于主内容区以作区分。
- **中间主工作区 (Main Calendar View)**
  - **核心区域**：展示类似 Notion Calendar 的时间网格。
  - **数据呈现**：用户的日常日程，以及 AI 自动生成的“待确认草稿”（以高亮或特殊边框显示）。
  - **交互**：支持拖拽调整时间，点击日程块弹出详情浮窗（如你提供的截图中展示的草稿确认浮窗）。
- **右侧边栏 (Right Panel - 任务与线索池) [可选/可折叠]**
  - 展示待分配的任务、AI 抓取到的原始房源列表或待办事项。

## 二、 核心 AI 交互流 (The Floating AI Assistant)

这是本次设计的灵魂。AI 入口不占据固定版面，而是采用**底部悬浮 (FAB - Floating Action Button) 结合气泡对话框**的模式。

### 状态 1：静默状态 (Idle State)

- **位置**：视窗底部居中，距离底边合适的留白（例如 `bottom-8` 或 `bottom-12`）。
- **形态**：一个精致的胶囊按钮或圆形 FAB，带有动效（如微弱的呼吸灯或流光描边），提示文案如“✨ 唤醒助理”或直接是一个图标。
- **层级**：`z-index` 极高，悬浮于日历网格之上。

### 状态 2：输入状态 (Input State)

- **触发**：用户点击底部的唤醒按钮。
- **动效**：按钮瞬间横向展开（建议使用 `framer-motion` 的 `layoutId` 实现平滑过渡），变成一个带发送按钮的输入框。
- **特性**：自动获取焦点 (`autoFocus`)，支持多行文本伸缩，支持快捷键（如 `Cmd + K` 直接唤醒，`Enter` 发送）。

### 状态 3：对话与工作流状态 (Active Dialog State)

- **位置与展开**：当用户输入并发送（或触发多轮对话时），对话框面板在**输入框正上方**平滑向上展开，呈现悬浮气泡形态。
- **UI 结构**：
  - **对话历史区**：展示用户的指令与 AI 的反问（如明确价格、提取风格）。
  - **卡片预览区**：在对话流中直接插入迷你版的日历卡片预览，实现所见即所得。
- **消失机制**：用户点击日历空白处（Click Outside）或按下 `Esc` 键，对话框收起，恢复为状态 1 的胶囊按钮，保证不遮挡日历主工作区。

## 三、 核心组件设计与 Tailwind CSS 结构思路

为了实现这种悬浮交互，我们需要一个高度封装的 `FloatingAgent` 组件，它独立于日历网格渲染。

### 1. `FloatingAgent` 容器组件

控制整体的定位与状态切换。

- **定位**：`fixed bottom-8 left-1/2 -translate-x-1/2 z-50`。
- **状态管理**：`isOpen` (控制是否展开为输入框), `chatHistory` (对话数组)。

### 2. `AgentInputBar` 组件

展开后的输入框，需要做到极致的跟手感。

- **样式**：`w-[600px] bg-white dark:bg-zinc-800 shadow-2xl rounded-2xl border border-zinc-200 p-2 flex items-center`。
- **阴影**：需要使用较重的阴影（如 `shadow-2xl`）将其从底层的日历网格中拔高。

### 3. `ChatOverlay` 组件

悬浮在输入框上方的气泡对话框。

- **定位关系**：使用绝对定位依附于输入框上方，如 `absolute bottom-[calc(100%+16px)] left-0 w-full`。
- **视觉约束**：设置最大高度 `max-h-[500px]` 配合 `overflow-y-auto`，确保对话记录过长时不会超出屏幕顶部；添加磨砂玻璃背景（`backdrop-blur-md`）增加质感。
- **渲染内容**：
  - 用户的普通文本消息。
  - AI 的多选项反问（如呈现几个按钮让用户选择“独栋”还是“公寓”）。
  - **行动卡片 (Action Cards)**：当 AI 构建好日程后，直接在气泡中渲染一个微缩版的日程块，用户点击“放入日历”，该日程块随即在背景的日历网格中显现。

## 四、 交互闭环示例（以初始化配置为例）

1. **用户打开网页**：看到的是极其干净的日历界面（此时可能是空的）。底部中间静静躺着一个“✨”按钮。
2. **点击/唤醒**：按钮展开为输入框。用户粘贴一大段原始需求：“每天早上九点帮我抓取...”
3. **气泡弹出**：输入框上方弹出悬浮面板。AI 回复：“收到，为了更精准，你需要关注哪类房产？”并在下方提供几个 Tag 供点击。
4. **无缝对接日历**：对话进行到第 3 轮，AI 回复：“配置已完成，我已经为你生成了今天的草稿。”
5. **魔法时刻**：随着这句回复，背景的日历网格 9:00 AM 处，**自动浮现并渲染出一个黄色的“待确认”日程块**。
6. **收起浮窗**：用户点击日历空白处，对话框和输入框收缩回到底部变成小按钮。用户直接在日历上点击刚才生成的日程块，在右侧（或弹窗）进行 30 秒确认。
